## Design Context

### Users
Korean professional targeting Big Tech-level English fluency. Uses the app throughout the day — at the office, messaging friends, writing to-do lists. Interactions are short and purposeful: open app, convert a phrase, copy, close. The job to be done is simple: **turn a Korean thought into natural, role-model-quality English as fast as possible.** The emotional goal: feel capable and confident in English, not overwhelmed or studied-at.

### Brand Personality
**Sharp · Vivid · Precise**

- **Sharp**: Clean white base, no noise. Typography-first layout. The interface steps back so the output steps forward.
- **Vivid**: Highlighter-colored accents — each mode has its own fluorescent color. Energetic without being chaotic.
- **Precise**: Every color placement is intentional — chip borders, active states, badges. Color as information, not decoration.

Voice and tone: direct and confident. Not playful, not clinical. Like a sharp assistant who highlights exactly what you need.

### Aesthetic Direction

**Visual tone**: Clean white base (light mode) / near-black base (dark mode). Fluorescent highlighter colors used as mode-coded accents — borders, active states, situation badges.

**Color palette**: 4-color fluorescent system, one per English register mode. White/black neutral base — neons only appear where they carry meaning.

| Token | Light | Dark |
|---|---|---|
| Background | `#FFFFFF` | `#0F0F0F` |
| Surface | `#F5F5F5` | `#1C1C1C` |
| Accent (primary / Speaking) | `#E8FF38` | `#E8FF38` |
| Accent (subtle) | `#F7FFBA` | `#272900` |
| Text (primary) | `#111111` | `#F2F2F2` |
| Text (secondary) | `#6B6B6B` | `#888888` |
| Border | `#E0E0E0` | `#2A2A2A` |

**Mode color system** (applied to chip borders, active states, situation badges):

| Mode | Color | Hex |
|---|---|---|
| Speaking | Neon yellow | `#E8FF38` |
| Messaging | Neon green | `#2AE87A` |
| Email | Hot pink | `#FF5FA0` |
| Writing | Electric blue | `#2ECFFF` |

**Typography**: Clean system sans-serif. Strong hierarchy: output expression dominant (22px/700), everything else supporting. Mode labels in uppercase.

**Anti-references**:
- Coral/terracotta "productivity app" warmth → replaced by clean white + fluorescent pop
- Duolingo / gamified color overload → highlighter colors appear only on mode-specific elements
- Purple-blue gradient AI slop → single-purpose neon colors, never as gradients

### Design Principles

1. **Shallow by default** — core action (convert) is always one tap from anywhere. Navigation never goes more than 2 levels deep.
2. **Speed is a feature** — input → output with zero unnecessary steps. No confirmation dialogs for reads.
3. **Output is the star** — the English expression is the most visually prominent element (22px/700). Everything else is supporting cast.
4. **Color = mode** — each fluorescent color maps to exactly one English register. Consistent across all screens. Never decorative.
5. **Neon on white, not everywhere** — fluorescent accents land on borders and active states only. The white base makes them pop without overwhelming.

## Component Notes

- **Inputs**: Large, easy-to-tap text areas (minHeight 100). Korean input should feel generous.
- **Output cards**: Expression dominant (22px/700). Situation label badge uses the mode's fluorescent color. "Tap to copy" top-right row alongside badge.
- **Sub-category chips**: Grouped by mode. Each chip border color matches its mode color. Active chip: filled with mode color + black text.
- **Convert button**: Neon yellow background (`#E8FF38`) + black text. Primary CTA.
- **Copy action**: Single tap on output card copies expression. "Copied" text feedback 1.5s.
- **Navigation**: Custom hamburger drawer (Animated + Modal). NOT bottom tabs.
- **Error states**: Distinguish network failure from AI failure. Never generic "Something went wrong" alone.
